# 老年人合理用药干预 · 药师工作台（离线版 V1.0）

配合《研究病例记录表》V2.0 与《附件2 老年人合理用药干预措施方案及标准化话术》使用的药师工具，
用于"基于药物素养的精准用药教育干预策略对老年慢性病患者的实施效果研究"项目（北京市卫健委老年人合理用药指导项目）。

## 功能

- **电子量表自动计分分层**：41 题药物素养量表 + AMRS 依从性量表，按 CRF 印制分值自动计分（含全部反向题自动赋值），自动计算 9 维度得分并按"≥4 良好 / 3~4 中等 / ≤3 不足"分层；
- **自动干预处方**：按分层生成干预重点、CRF ①-⑧ 措施预勾选、薄弱维度教育要点、对应 B1/B2/B3 话术一键复制；
- **随访管理**：自动计算第 1 / 3 个月随访窗口（±7 天）并提醒，随访表单按 CRF 条目顺序引导，3 个月随访含量表复测（自动对比基线）与满意度；
- **话术库**：入组、首次干预、微信群运营（9 期主题推送 + 答疑 + 转诊）、随访全套 32 条标准化话术，一键复制；
- **数据导出**：患者汇总 CSV（可导入 SPSS）、量表逐题明细 CSV、JSON 备份/恢复。

## 云端加密备份（选配）

除本机 JSON 备份外，可在【设置 → 云端加密备份】开启数据自动加密上传到项目组指定的 **Gitee 私有仓库**（境内），用于异地容灾与换设备恢复。

- **端到端加密**：数据先在本机用加密口令加密（AES-256-GCM），云端只存密文，无口令任何人（含仓库管理员）都无法读取；令牌与口令只存本机，不写入网页代码，也不含在导出文件中。
- **配置三项**：数据仓库（如 `用户名/yaoshi-data`，须私有库）、Gitee 私人令牌（仅勾 `projects` 权限）、加密口令（全项目统一）；勾选自动上传 → 保存配置 → 立即上传测试。
- **换设备恢复**：新设备填入相同的仓库/令牌/口令 → 点【从云端恢复】即可还原。

> ⚠️ 各分中心先在「分中心信息」填本中心「中心号」，备份文件才会按中心分开命名（`center-01…`）不互相覆盖。全项目使用**同一加密口令**，务必安全下发并归档——口令丢失则云端备份永久无法解密。令牌/口令属敏感凭据，请勿发患者群或写入任何公开文件。

## 数据安全

默认情况下，数据仅保存在使用者本机浏览器（localStorage），不上传任何服务器；系统内仅录入受试者编号与姓名缩写，不含直接身份信息。开启上述云端备份后，离开本机的仅为加密密文，且仓库为私有。请按使用说明定期备份。
